Migration Sizing Information

About Mainframe Migration Sizing

Migration sizing uses the Workload Factor (WLF) approach that converts MIPS to transactions per minute (tpm), then determines the required processor cores based on tpm capacity. This methodology accounts for different workload characteristics, providing more accurate sizing than simple MIPS comparisons.

Workload Factor (WLF) Values

  • 30 tpm/MIPS: Processor-intensive workloads (scientific computing, simulations)
  • 75 tpm/MIPS: TPC-C workloads (SAP SD)
  • 120 tpm/MIPS: Commercial workloads (SAP ERP, Oracle EBS)
  • 160 tpm/MIPS: Mixed workloads (SAP BW)
  • 200 tpm/MIPS: Business intelligence workloads (data warehousing)

Conversion Formula

Required MIPS = Utilization × Current tpm / WLF

Where:

  • Utilization: Current mainframe utilization percentage
  • Current tpm: Transaction throughput (calculated from MIPS × WLF)
  • WLF: Workload Factor based on workload type

Validated MIPS per Core Values

  • Intel Xeon (3.0 GHz Quad Core): 271 MIPS per core
  • Intel Itanium (1.6 GHz Dual Core): 295 MIPS per core

Architecture Comparison

IBM Mainframe Virtualization

PR/SM LPAR

IBM's Processor Resource/System Manager provides hardware-level virtualization with extremely low overhead.

Strengths:

  • Hardware-enforced isolation
  • Dedicated I/O processors
  • Very low overhead (typically <5%)
  • Strong workload isolation
  • Mature hypervisor technology
  • Designed for mission-critical workloads

Weaknesses:

  • Higher cost
  • Proprietary architecture
  • Limited ecosystem

IBM POWER Virtualization

PowerVM

IBM's virtualization technology for POWER systems with micro-partitioning and shared processor pools.

Strengths:

  • Micro-partitioning (down to 0.01 of a core)
  • Live partition mobility
  • Dynamic resource allocation
  • SMT-8 (8 threads per core) for dense virtualization
  • Strong isolation
  • Mature hypervisor with mainframe-inspired features

Weaknesses:

  • Higher cost than x86 virtualization
  • Smaller ecosystem than x86
  • Specialized skills required

Intel x86 Virtualization

Various (VMware, KVM, Hyper-V, Xen)

x86 virtualization through hardware-assisted hypervisors.

Strengths:

  • Broad ecosystem of virtualization solutions
  • Good performance with hardware-assisted virtualization
  • Cost-effective virtualization for general workloads
  • Familiar management tools
  • Wide range of supported guest operating systems

Weaknesses:

  • Higher overhead than POWER or mainframe
  • Less isolation between workloads
  • Less predictable performance under heavy load

IBM Mainframe Network Architecture

IBM mainframes feature integrated networking with dedicated I/O processors and channels.

Components:

  • Integrated Hardware Cryptographic Acceleration
  • OSA-Express (Open Systems Adapter)
  • HiperSockets (high-speed in-memory network)
  • Shared Memory Communications (SMC)
  • z/OS Communications Server

Strengths:

  • Extremely high throughput
  • Very low latency
  • Hardware encryption offload
  • High reliability and redundancy
  • Integrated into the system architecture

IBM POWER Network Architecture

POWER systems utilize high-bandwidth I/O subsystems with PCIe connectivity.

Components:

  • PowerVM Virtual I/O Server (VIOS)
  • Shared Ethernet Adapters (SEA)
  • SR-IOV capable adapters
  • RoCE (RDMA over Converged Ethernet)
  • PowerVM logical partitioning of network resources

Strengths:

  • High bandwidth
  • Virtualized I/O with minimal overhead
  • SR-IOV support for direct hardware access
  • Good isolation between partitions
  • Flexible networking options

Intel x86 Network Architecture

Intel-based systems use standard PCIe networking with various virtualization technologies.

Components:

  • Standard PCIe network adapters
  • SR-IOV capable adapters
  • DPDK (Data Plane Development Kit)
  • Virtualized networking via hypervisors
  • Software-defined networking options

Strengths:

  • Wide range of networking options
  • Cost-effective solutions
  • Good performance with specialized adapters
  • Flexible configuration options
  • Strong ecosystem support

IBM Mainframe Architecture

IBM POWER Architecture

Partitioning Technology:

PowerVM LPAR

Hypervisor-based logical partitioning with micro-partitioning capabilities.

Virtualization Technology:

Cloud Technologies:

Cloud Technologies:

  • Familiar management tools

About This Tool

Purpose

This comparison tool is designed to help organizations understand the performance characteristics and migration options for different processor architectures. It focuses on IBM POWER, Intel x86, and IBM Mainframe systems, providing insights into their relative performance for various workload types.

Methodology

The comparison methodology used in this tool is based on:

  • Workload Factor (WLF) approach: Converts MIPS to transactions per minute (tpm), then determines the required processor cores based on tpm capacity.
  • Validated performance metrics: Uses performance data from testing by Micro Focus and IBM's own methodologies.
  • Workload-specific performance: Accounts for different performance characteristics based on workload type.

Key Features

  • Processor comparison with detailed specifications
  • Mainframe migration sizing calculator
  • Architecture comparison for virtualization, networking, and cloud capabilities
  • Workload-specific performance analysis
  • Visual comparison through charts and metrics

Data Sources

The data used in this tool comes from:

  • IBM's Large System Performance Reference (LSPR)
  • Micro Focus testing on HP Integrity and HP ProLiant servers
  • Published processor specifications from IBM and Intel
  • Industry standard benchmarks and performance metrics
  • HP Sizing Mainframe Workloads research paper
